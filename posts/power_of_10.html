NASA’s Power of 10 Rules

Writing software that must work — every time.

When software controls spacecraft, power systems, or industrial machinery, failure is not an option. To reduce catastrophic risk, NASA’s Jet Propulsion Laboratory introduced the Power of 10 Rules — a set of constraints designed to make software predictable, verifiable, and safe.

These rules originated in embedded C, but the philosophy applies equally to modern Python services, ML pipelines, and distributed systems.

1️⃣ No Dynamic Memory Allocation After Initialization

Dynamic memory can fail at runtime and fragment memory. NASA requires all memory to be allocated once, up front.

❌ Python (risky)


def process(data):
    buffer = []
    for x in data:
        buffer.append(x * 2)
    return buffer

✅ Python (bounded & predictable)


MAX_ITEMS = 100

def process(data):
    buffer = [0] * MAX_ITEMS
    for i, x in enumerate(data[:MAX_ITEMS]):
        buffer[i] = x * 2
    return buffer

✅ C++ (pre-allocated)


constexpr int MAX_ITEMS = 100;
int buffer[MAX_ITEMS];

void process(const int* data, int length) {
    for (int i = 0; i < length && i < MAX_ITEMS; i++) {
        buffer[i] = data[i] * 2;
    }
}

3️⃣ All Loops Must Have Fixed Upper Bounds

Infinite or data-dependent loops are unacceptable in safety-critical systems. Worst-case execution time must always be known.

❌ Dangerous loop


while sensor.is_active():
    read_sensor()

✅ Bounded loop


MAX_READS = 50

for _ in range(MAX_READS):
    if not sensor.is_active():
        break
    read_sensor()

✅ C++ equivalent


for (int i = 0; i < 50; i++) {
    if (!sensor_active()) break;
    read_sensor();
}

7️⃣ Always Check Return Values

Ignoring return values hides failure. Every operation must be verified.

❌ Ignored error


send_packet(packet)

✅ Explicit handling


status = send_packet(packet)
if status != OK:
    log_error(status)
    shutdown_system()

✅ C++


Status status = send_packet(packet);
if (status != OK) {
    log_error(status);
    shutdown_system();
}

Why These Rules Still Matter

Whether you're building spacecraft software, telemetry pipelines, or ML-backed monitoring systems, the lesson is the same:

Predictability beats cleverness.
If you can’t reason about failure, you can’t trust the system.

NASA’s Power of 10 Rules are not about writing less powerful software — they’re about writing software that survives the real world.
