Cyclone Classification ML Project

Predicting tropical cyclone categories from meteorological data using supervised learning.

Overview

This project focuses on classifying tropical cyclones into intensity categories based on official meteorological records. It was developed as the final project for a university Machine Learning course and demonstrates the full workflow: data exploration, preprocessing, model training, evaluation, and interpretation.

The goal is to learn from historical cyclone data and build a model that can accurately assign each observation to the correct cyclone type or intensity level, supporting better understanding and potential decision support for extreme weather analysis.

Dataset

  • Official tropical cyclone dataset with labeled cyclone categories.
  • Tabular features such as intensity-related measurements and meteorological variables.
  • Data cleaning: handled missing values, removed inconsistencies, and standardized formats.
  • Train / test split to evaluate generalization performance on unseen data.

All analysis and experiments are contained inside a Jupyter Notebook (final_project.ipynb) in the project repository.

Modeling Approach

The project follows a classic supervised learning pipeline for classification:

  • Exploratory Data Analysis (EDA) to understand feature distributions and class balance.
  • Feature preprocessing (scaling / encoding as needed, depending on feature types).
  • Training several classification models (e.g. logistic regression, tree-based methods) and comparing them.
  • Hyperparameter tuning using cross-validation to improve performance.
  • Evaluating models with metrics appropriate for multi-class classification.

The final selected model strikes a balance between accuracy, robustness, and interpretability.

Results & Evaluation

Model performance is evaluated using:

  • Accuracy on the held-out test set.
  • Confusion matrix to understand per-class performance and misclassifications.
  • Precision, recall, and F1-score to account for potential class imbalance.

Visualizations include:

  • Class distribution plots.
  • Confusion matrix heatmap.
  • Feature importance / contribution analysis (where applicable).

Together, these provide a clear picture of how well the model distinguishes between cyclone categories and where it can be improved with more data or additional features.

Tech Stack

  • Language: Python
  • Environment: Jupyter Notebook
  • Libraries: NumPy, Pandas, Scikit-Learn, Matplotlib / Seaborn
  • Version control: Git & GitHub
