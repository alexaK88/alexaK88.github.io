Autonomous Diver Companion

Diver Wearable Safety Simulator

A Python-based simulator for the wearable-side safety pipeline of an autonomous underwater diver companion system. The project models simulated diver telemetry, preprocessing, state estimation, and local alarm generation.

← Back to Projects GitHub Repo →

Overview

This project represents the wearable-side component of a larger autonomous diver companion concept. The current module simulates a diver-worn device that produces raw local telemetry, processes the data into safety-relevant signals, estimates the diver's current state, and generates structured safety packets.

The focus is on clean backend architecture, typed data models, modular processing stages, and safety-oriented rule evaluation. The system is designed as a foundation for later integration with a companion robot, surface gateway, backend service, and operator dashboard.

Problem & Goals

  • Problem: Underwater divers may face safety-critical situations such as fast ascent, low gas, lost communication, no motion, or explicit emergency conditions.
  • Goal: Build a modular wearable-side simulator that converts raw diver telemetry into structured safety information.
  • Outcome: A first working vertical slice of the wearable safety pipeline: raw packets → preprocessing → state estimation → alarm evaluation.

Current Pipeline

The current wearable-side flow is:

DiverWearableSimulator
    ↓
RawWearablePacket
    ↓
SignalPreprocessor
    ↓
CleanSensorPacket
    ↓
DiverStateEstimator
    ↓
DiverState
    ↓
AlarmEngine
    ↓
DiverSafetyPacket

Each layer has a separate responsibility. The simulator generates raw measurements, the preprocessor calculates rates, the estimator interprets the diver state, and the alarm engine decides whether the interpreted state is safety-relevant.

Implemented Features

  • Scenario-based diver wearable simulation
  • Depth, tank pressure, compass heading, motion, battery, button, and link-quality signals
  • Time-based simulation of gas pressure, battery drain, compass heading, and dive profile
  • Preprocessing layer for ascent/descent rate and gas pressure drop rate
  • Diver state estimation for dive phase, motion state, gas state, and link state
  • Rule-based alarm engine for safety-relevant conditions
  • Emergency button propagation through the full pipeline
  • Pytest tests for preprocessing, scenarios, compass behaviour, and end-to-end pipeline logic

Supported Scenarios

  • NORMAL: baseline simulated dive behaviour
  • LOW_GAS: low tank pressure scenario
  • FAST_ASCENT: unsafe ascent rate scenario
  • EMERGENCY_BUTTON: diver-triggered emergency signal
  • NO_MOTION: reduced motion signal scenario
  • WEAK_LINK: degraded wearable communication link
  • LOST_LINK: near-zero communication quality
  • BATTERY_LOW: wearable battery warning scenario

Safety Logic

The alarm engine evaluates interpreted diver states and generates alarm events when a condition becomes safety-relevant. Current alarms include fast ascent, low or critical gas, unusually fast pressure drop, weak or lost link, low wearable battery, no-motion conditions, and emergency button activation.

The final output is a DiverSafetyPacket, which contains the original interpreted diver state, all detected alarms, and an overall safety state: 

OK
WARNING
CRITICAL
EMERGENCY

Current Status

The project is currently at the first wearable-side milestone. The local simulation pipeline is implemented and tested, including raw telemetry generation, preprocessing, state estimation, and alarm evaluation.

The next planned step is to design the robot-side system: packet reception, diver tracking, robot mission state estimation, communication loss handling, and behaviour selection.

Future Work

  • Packet builder and transmitter for wearable-to-robot communication
  • Communication simulator with packet loss, weak link, and stale packet handling
  • Robot-side diver tracker and mission state estimator
  • Robot behaviour controller for guidance, search, and emergency support modes
  • Surface gateway and backend integration
  • Replay/logging system for scenario analysis
  • Operator dashboard for live safety monitoring
