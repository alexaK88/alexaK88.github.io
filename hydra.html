HYDRA-7

Underwater Object and Anomaly Detection System

Project Overview

HYDRA-7 is a vehicle-agnostic underwater perception system designed to process live or recorded camera streams from underwater platforms such as ROVs, AUVs, submersibles, inspection cameras, or simulated mission feeds.

The system combines a YOLO-based underwater object detection model with a near-real-time video processing runtime. It detects relevant objects in underwater video, overlays bounding boxes, saves detection events, and exposes runtime state for a future operator dashboard.

The current version focuses on building the complete perception system around the model: stream handling, threaded inference, event storage, logging, and backend-ready architecture.

Current Status

The project currently includes a trained baseline object detection model and a threaded video inference pipeline. The runtime processes video close to real time by separating frame reading, inference, and display logic.

  • Baseline YOLO model trained on underwater object data
  • Near-real-time video inference prototype
  • Threaded runtime architecture for smoother stream processing
  • Detection overlays on underwater video
  • Automatic event saving for detected objects
  • Raw frame, annotated frame, and metadata storage
  • Runtime performance logging

Operator Interface Concept

The planned dashboard is designed as an operator-facing console for live underwater perception. It will display the annotated camera stream, active detections, alert history, runtime FPS, target classes, saved captures, and system status.

System Architecture

HYDRA-7 is designed as a modular system rather than a single detection script. The perception runtime separates stream input, frame processing, object detection, decision logic, runtime state, storage, and dashboard communication.

This structure allows the same system to work with different input sources. The first version uses video files and webcam input, while future versions can support RTSP streams, ROS 2 image topics, AUV cameras, or edge-deployed perception nodes.

Technical Implementation

Perception Runtime

A threaded video pipeline separates frame reading from model inference, allowing the video stream to remain responsive while detections are updated asynchronously.

Object Detection

The current baseline uses a YOLO model trained on underwater imagery. The first detection scope includes marine animals such as fish, crabs, jellyfish, shrimp, and starfish.

Event Storage

When detections occur, the system saves raw frames, annotated frames, and metadata JSON files for later review, debugging, and future dataset expansion.

Dashboard-Ready Backend

The system is being structured around a FastAPI backend exposing runtime status, latest detections, saved events, and an MJPEG video stream for a web dashboard.

Tech Stack

The current implementation is Python-first, with a planned C++ runtime layer for optimized edge or robotics deployment.

  • Python 3.11 for system runtime and backend development
  • uv for environment and dependency management
  • OpenCV for video processing and frame annotation
  • Ultralytics YOLO for underwater object detection
  • PyTorch for current model inference
  • FastAPI + Uvicorn for the backend service
  • JSON / JSONL for event metadata and logs
  • React + TypeScript planned for the dashboard
  • C++ / ONNX Runtime planned for optimized inference

Engineering Focus

The main challenge is not only training a model, but building the surrounding system that makes underwater perception usable in a live mission context.

HYDRA-7 focuses on the engineering layers around AI perception: live stream processing, runtime state, event-driven storage, API design, system observability, and future deployment constraints.

This makes the project closer to an underwater robotics perception platform than a standalone computer vision notebook.

Roadmap

  • Convert the threaded prototype into a FastAPI backend runtime
  • Add live MJPEG stream endpoint
  • Build the first HYDRA-7 dashboard view
  • Add configurable target profiles
  • Add image-quality anomaly checks for blur, low light, and low contrast
  • Add SQLite-based searchable event history
  • Expand the model beyond marine animals to debris, divers, wreckage, and underwater vehicles
  • Export the model to ONNX
  • Implement a C++ perception runtime for optimized deployment
