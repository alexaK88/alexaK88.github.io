Two Sum

Easy

Pattern: Hash map / complement lookup.

The simple way would be to check every pair of numbers, but that would be slow. Instead, I keep a hash map of numbers I have already seen. For each number, I calculate the value needed to reach the target. If that needed value is already in the hash map, I found the answer.

For example, if the target is 9 and the current number is 2, I need 7. So I check whether 7 was already seen.

Complexity: O(n) time, O(n) space.

class Solution:
          def twoSum(self, nums: list[int], target: int) -> list[int]:
              seen = {}
      
              for i, num in enumerate(nums):
                  complement = target - num
      
                  if complement in seen:
                      return [seen[complement], i]
      
                  seen[num] = i

What I learned: A hash map lets me avoid checking every possible pair. Instead of asking “which two numbers add up to the target?”, I ask “what number do I need to complete the pair?”

Valid Anagram

Easy

Pattern: Hash map / frequency counting.

Two strings are anagrams if they contain the same characters with the same frequencies. For example, "anagram" and "nagaram" are anagrams because every character appears the same number of times in both strings.

Approach 1: Pythonic solution with Counter

The shortest and cleanest solution is to use Counter. It creates a frequency map for each string, then compares the two maps directly. 

from collections import Counter
      
      class Solution:
          def isAnagram(self, s: str, t: str) -> bool:
              return Counter(s) == Counter(t)

This version is very readable because the code directly says what we mean: count the characters in both strings and compare the counts.

Approach 2: Manual hash map solution

The manual solution does the same thing, but without using Counter. First, I count all characters in the first string. Then, while scanning the second string, I subtract from those counts. If a character is missing or used too many times, the strings cannot be anagrams. 

class Solution:
          def isAnagram(self, s: str, t: str) -> bool:
              if len(s) != len(t):
                  return False
      
              counts = {}
      
              for ch in s:
                  if ch not in counts:
                      counts[ch] = 1
                  else:
                      counts[ch] += 1
      
              for ch in t:
                  if ch not in counts or counts[ch] == 0:
                      return False
                  counts[ch] -= 1
      
              return True

Complexity: O(n) time. The space complexity is O(1) if the alphabet is fixed, for example only lowercase English letters, and O(k) in the general case, where k is the number of unique characters.

What I learned: Counter is great for clean Python code, but the manual hash map version is useful because it shows the actual logic behind frequency counting.

Contains Duplicate

Easy

Pattern: Set / hash-based lookup.

The task is to check whether any value appears more than once in the array. The key idea is that sets and hash maps can tell us very quickly whether we have already seen a value before.

Approach 1: Set length comparison

A set only stores unique values. So if the set is smaller than the original list, that means at least one value appeared more than once. 

class Solution:
          def containsDuplicate(self, nums: list[int]) -> bool:
              unique_values = set(nums)
              return len(unique_values) < len(nums)

This is the cleanest Python solution because it directly uses the property of a set: duplicates are automatically removed.

Approach 2: Manual hash map solution

Another way is to scan through the array and store every number I have already seen. If I reach a number that is already in the hash map, I immediately know there is a duplicate. 

class Solution:
          def containsDuplicate(self, nums: list[int]) -> bool:
              seen = {}
      
              for num in nums:
                  if num in seen:
                      return True
      
                  seen[num] = 1
      
              return False

This version is slightly more verbose, but it is useful because it shows the general hash-map idea used in many array problems: store what has already appeared, then check future values against it.

Complexity: O(n) time and O(n) space for both approaches.

What I learned: If the problem only asks whether duplicates exist, a set is usually enough. A hash map becomes more useful when I also need counts, positions, or extra information about each value.

Group Anagrams

Medium

Pattern: Hash map / grouping by a shared key.

The goal is to group words that are anagrams of each other. Anagrams contain the same characters, only in a different order. The main idea is to create a key that is identical for all anagrams.

My first working version

My first solution used a dictionary where the key was the sorted version of each word. If two words have the same sorted form, they belong in the same group. 

class Solution:
          def groupAnagrams(self, strs: list[str]) -> list[list[str]]:
              hash_map = {}
              result = []
      
              for word in strs:
                  sorted_word = ''.join(sorted(word))
      
                  if sorted_word not in hash_map.keys():
                      hash_map[sorted_word] = [word]
                  else:
                      hash_map[sorted_word].append(word)
      
              for key in hash_map.keys():
                  result.append(hash_map[key])
      
              return result

This works correctly. For example, "eat", "tea", and "ate" all become "aet" after sorting, so they are stored under the same dictionary key.

Improvement 1: Avoid using .keys() for membership checks

Instead of writing sorted_word not in hash_map.keys(), I can write sorted_word not in hash_map. It means the same thing, but it is cleaner and more Pythonic. 

class Solution:
          def groupAnagrams(self, strs: list[str]) -> list[list[str]]:
              hash_map = {}
              result = []
      
              for word in strs:
                  sorted_word = ''.join(sorted(word))
      
                  if sorted_word not in hash_map:
                      hash_map[sorted_word] = [word]
                  else:
                      hash_map[sorted_word].append(word)
      
              for key in hash_map:
                  result.append(hash_map[key])
      
              return result

Improvement 2: Return dictionary values directly

The dictionary already stores the final groups as values. So I do not need to create a separate result list manually. I can return list(hash_map.values()). 

class Solution:
          def groupAnagrams(self, strs: list[str]) -> list[list[str]]:
              hash_map = {}
      
              for word in strs:
                  sorted_word = ''.join(sorted(word))
      
                  if sorted_word not in hash_map:
                      hash_map[sorted_word] = [word]
                  else:
                      hash_map[sorted_word].append(word)
      
              return list(hash_map.values())

Improvement 3: Use defaultdict

With defaultdict(list), I do not need to manually check whether the key already exists. If the key is new, Python automatically creates an empty list for it. 

from collections import defaultdict
      
      class Solution:
          def groupAnagrams(self, strs: list[str]) -> list[list[str]]:
              groups = defaultdict(list)
      
              for word in strs:
                  key = ''.join(sorted(word))
                  groups[key].append(word)
      
              return list(groups.values())

This is the cleanest version of the sorted-key approach.

Optional improvement: Character-count key

Sorting each word works well, but sorting costs extra time. Another approach is to count how many times each letter appears and use that count as the key. 

from collections import defaultdict
      
      class Solution:
          def groupAnagrams(self, strs: list[str]) -> list[list[str]]:
              groups = defaultdict(list)
      
              for word in strs:
                  counts = [0] * 26
      
                  for ch in word:
                      counts[ord(ch) - ord('a')] += 1
      
                  groups[tuple(counts)].append(word)
      
              return list(groups.values())

This avoids sorting, but it assumes the input contains only lowercase English letters. The sorted-key version is easier to read and more general.

Complexity: The sorted-key solution takes O(n · k log k) time, where n is the number of words and k is the maximum word length. The character-count solution takes O(n · k) time. Both use O(n · k) space to store the groups.

What I learned: The important part of grouping problems is choosing a good key. Here, sorting each word creates the same key for all anagrams. Then the hash map does the grouping for me.

Top K Frequent Elements

Medium

Pattern: Hash map / frequency counting.

The goal is to return the k numbers that appear most often in the array. The first step is to count how many times each number appears. After that, I need to extract the numbers with the highest frequencies.

My first working version

My first solution used Counter, which is a clean Python tool for counting frequencies. Then I used most_common(k) to get the k most frequent elements. 

from collections import Counter
        from typing import List
        
        class Solution:
            def topKFrequent(self, nums: List[int], k: int) -> List[int]:
                list_most_common = []
                cnt = Counter(nums)
        
                for i, (key, frequency) in enumerate(cnt.most_common(k)):
                    list_most_common.append(key)
        
                return list_most_common

This works correctly. Counter(nums) creates a frequency map, and most_common(k) returns the most frequent values together with their counts.

Improvement 1: Remove unused variables

In my first version, I used enumerate, but I did not actually need the index. I also did not need to store the frequency because the final answer only asks for the numbers. 

from collections import Counter
        
        class Solution:
            def topKFrequent(self, nums: list[int], k: int) -> list[int]:
                result = []
                counts = Counter(nums)
        
                for num, freq in counts.most_common(k):
                    result.append(num)
        
                return result

Improvement 2: Use a list comprehension

Since I only want the numbers from most_common(k), I can make the code even shorter with a list comprehension. 

from collections import Counter
        
        class Solution:
            def topKFrequent(self, nums: list[int], k: int) -> list[int]:
                counts = Counter(nums)
                return [num for num, freq in counts.most_common(k)]

This is the cleanest Pythonic version. It is short, readable, and uses the standard library well.

Interview-style improvement: Bucket sort

The Counter solution is clean, but for interviews it is useful to know the bucket-sort idea too. The maximum possible frequency is len(nums), so I can create buckets where the index represents the frequency.

For example, if a number appears 3 times, I put it into bucket 3. Then I scan the buckets from highest frequency to lowest frequency until I collect k numbers. 

from collections import Counter
        
        class Solution:
            def topKFrequent(self, nums: list[int], k: int) -> list[int]:
                counts = Counter(nums)
        
                buckets = [[] for _ in range(len(nums) + 1)]
        
                for num, freq in counts.items():
                    buckets[freq].append(num)
        
                result = []
        
                for freq in range(len(buckets) - 1, 0, -1):
                    for num in buckets[freq]:
                        result.append(num)
        
                        if len(result) == k:
                            return result

This version avoids sorting the numbers by frequency. Instead, it uses frequency as an index.

Complexity: The Counter version is very clean, but the internal work is hidden inside most_common. The bucket-sort version has O(n) time and O(n) space, because it counts the numbers, places them into buckets, and scans the buckets once.

What I learned: Counter is excellent for writing clean Python code, but the bucket-sort approach shows the deeper algorithmic idea: when frequencies are bounded by the array length, I can use the frequency itself as a position in an array.

Sales by Match / Sock Merchant

Easy

Pattern: Hash map / frequency counting.

The goal is to count how many matching pairs of socks exist. Each number represents a sock colour. If a colour appears 5 times, it forms 2 pairs, because one sock is left without a match.

My first working version

My first solution uses a dictionary to count how many times each sock colour appears. After counting, I go through the dictionary and add count // 2 for each colour. 

def sockMerchant(n, ar):
              hash_map = {}
              total = 0
          
              for i in range(n):
                  if ar[i] in hash_map.keys():
                      hash_map[ar[i]] += 1
                  else:
                      hash_map[ar[i]] = 1
          
              for key in hash_map.keys():
                  total += hash_map[key] // 2
          
              return total

This works because integer division by 2 gives the number of full pairs for each colour. For example, if a colour appears 5 times, then 5 // 2 = 2 pairs.

Improvement 1: Avoid using .keys()

Instead of writing ar[i] in hash_map.keys(), I can write ar[i] in hash_map. It means the same thing, but it is cleaner. 

def sockMerchant(n, ar):
              hash_map = {}
              total = 0
          
              for i in range(n):
                  if ar[i] in hash_map:
                      hash_map[ar[i]] += 1
                  else:
                      hash_map[ar[i]] = 1
          
              for key in hash_map:
                  total += hash_map[key] // 2
          
              return total

Improvement 2: Iterate directly over the socks

I do not actually need the index i. I only need each sock colour, so I can loop directly over ar. 

def sockMerchant(n, ar):
              counts = {}
              total = 0
          
              for sock in ar:
                  if sock in counts:
                      counts[sock] += 1
                  else:
                      counts[sock] = 1
          
              for sock in counts:
                  total += counts[sock] // 2
          
              return total

Improvement 3: Use dict.get()

With dict.get(), I can avoid writing the full if/else block when counting. 

def sockMerchant(n, ar):
              counts = {}
          
              for sock in ar:
                  counts[sock] = counts.get(sock, 0) + 1
          
              total = 0
          
              for count in counts.values():
                  total += count // 2
          
              return total

Improvement 4: Use Counter

Since this is a frequency-counting problem, Counter gives a very clean Python solution. 

from collections import Counter
          
          def sockMerchant(n, ar):
              counts = Counter(ar)
              return sum(count // 2 for count in counts.values())

This is the cleanest version. It directly says: count the socks, then sum the number of pairs for each colour.

Optional improvement: Count pairs while scanning

Another approach is to count pairs immediately. I keep a set of unmatched socks. When I see a sock colour for the first time, I store it. When I see the same colour again, I remove it and count one pair. 

def sockMerchant(n, ar):
              unmatched = set()
              pairs = 0
          
              for sock in ar:
                  if sock in unmatched:
                      pairs += 1
                      unmatched.remove(sock)
                  else:
                      unmatched.add(sock)
          
              return pairs

This version is nice because it does not need to store full counts. It only tracks socks waiting for a match.

Complexity: All versions take O(n) time. The dictionary and Counter versions use O(k) space, where k is the number of unique sock colours. The unmatched-set version also uses O(k) space.

What I learned: This is a frequency-counting problem. A dictionary is enough, but Counter makes the counting cleaner. The set approach is also useful because it counts pairs as they appear instead of counting everything first.

Product of Array Except Self

Medium

Pattern: Prefix product / suffix product.

The goal is to build an array where each position contains the product of all numbers except the number at that position. The tricky part is doing this without using division.

My first working idea: total product with zero handling

My first solution calculated the product of all non-zero numbers and then handled three cases: no zeros, exactly one zero, and more than one zero. 

class Solution:
              def productExceptSelf(self, nums: list[int]) -> list[int]:
                  product = 1
                  has_zeros = False
                  exactly_one_zero = False
                  n = len(nums)
          
                  for el in nums:
                      if el == 0:
                          if not exactly_one_zero and has_zeros:
                              continue
                          elif exactly_one_zero:
                              exactly_one_zero = False
                          else:
                              has_zeros = True
                              exactly_one_zero = True
                      else:
                          product *= el
          
                  if has_zeros and not exactly_one_zero:
                      return [0] * n
          
                  answer = [0] * n
          
                  for i in range(n):
                      if exactly_one_zero:
                          if nums[i] == 0:
                              answer[i] = product
                      else:
                          answer[i] = product // nums[i]
          
                  return answer

This works by separating the zero cases. If there are two or more zeros, every product except self contains at least one zero, so the whole answer is zero. If there is exactly one zero, only the zero position gets the product of all non-zero values.

Improvement 1: Simplify the zero-count logic

Instead of using two boolean variables, I can count how many zeros there are. This makes the logic easier to read. 

class Solution:
              def productExceptSelf(self, nums: list[int]) -> list[int]:
                  zero_count = nums.count(0)
                  n = len(nums)
          
                  if zero_count > 1:
                      return [0] * n
          
                  product = 1
                  for num in nums:
                      if num != 0:
                          product *= num
          
                  answer = [0] * n
          
                  if zero_count == 1:
                      for i, num in enumerate(nums):
                          if num == 0:
                              answer[i] = product
                      return answer
          
                  for i, num in enumerate(nums):
                      answer[i] = product // num
          
                  return answer

This version is cleaner, but it still uses division. That means it does not fully satisfy the intended constraint of the problem.

Improvement 2: Prefix and suffix products

The better idea is to avoid division completely. For each position, I need the product of everything on the left and everything on the right.

For example, for index i: 

answer[i] = product of nums before i * product of nums after i

So I can first build prefix products from the left, then multiply by suffix products from the right. 

class Solution:
              def productExceptSelf(self, nums: list[int]) -> list[int]:
                  n = len(nums)
          
                  prefix = [1] * n
                  suffix = [1] * n
                  answer = [1] * n
          
                  for i in range(1, n):
                      prefix[i] = prefix[i - 1] * nums[i - 1]
          
                  for i in range(n - 2, -1, -1):
                      suffix[i] = suffix[i + 1] * nums[i + 1]
          
                  for i in range(n):
                      answer[i] = prefix[i] * suffix[i]
          
                  return answer

This version works even when the array contains zeros, because it never divides by anything.

Improvement 3: Optimised prefix/suffix solution

I do not actually need separate prefix and suffix arrays. I can store prefix products directly in the answer array, then multiply by suffix products while scanning from the right. 

class Solution:
              def productExceptSelf(self, nums: list[int]) -> list[int]:
                  n = len(nums)
                  answer = [1] * n
          
                  prefix_product = 1
                  for i in range(n):
                      answer[i] = prefix_product
                      prefix_product *= nums[i]
          
                  suffix_product = 1
                  for i in range(n - 1, -1, -1):
                      answer[i] *= suffix_product
                      suffix_product *= nums[i]
          
                  return answer

In the first pass, answer[i] stores the product of everything to the left of i. In the second pass, I multiply it by the product of everything to the right.

Complexity: The final solution takes O(n) time. It uses O(1) extra space if the output array is not counted as extra space.

What I learned: My first instinct was to calculate the total product and divide, but the stronger solution is to split the problem into left-side products and right-side products. Prefix and suffix thinking removes the need for division and automatically handles zeros.

Range Sum Query - Immutable

Easy

Pattern: Prefix sum.

The goal is to return the sum of elements between two indices, left and right, many times. The array does not change, so it makes sense to do some preprocessing once and make every later query faster.

My first working version

My first solution stored a copy of the input array. Every time sumRange was called, I looped from left to right and added the values manually. 

class NumArray:
          
              def __init__(self, nums: list[int]):
                  self.array = nums.copy()
          
              def sumRange(self, left: int, right: int) -> int:
                  total = 0
          
                  for i in range(left, right + 1):
                      total += self.array[i]
          
                  return total

This works correctly. The problem is that if the range is large, or if sumRange is called many times, I keep repeating the same addition work again and again.

Complexity of my first version: O(n) time per query in the worst case, and O(n) space because I store a copy of the array.

Improvement: Prefix sum array

A prefix sum array stores the running total up to each position. Then a range sum can be calculated by subtracting two prefix sums.

I use one extra starting value, 0, so the prefix array is one element longer than the original array. This makes the formula cleaner. 

class NumArray:
          
              def __init__(self, nums: list[int]):
                  self.prefix = [0]
          
                  for num in nums:
                      self.prefix.append(self.prefix[-1] + num)
          
              def sumRange(self, left: int, right: int) -> int:
                  return self.prefix[right + 1] - self.prefix[left]

For example, if I want the sum from index left to index right, I take the total up to right and subtract the total before left. 

sum(left, right) = prefix[right + 1] - prefix[left]

This gives the answer immediately, without looping through the range again.

Complexity of the improved version: O(n) time during initialization, O(1) time per query, and O(n) space for the prefix sum array.

What I learned: If the same kind of range query is asked many times, it is often worth preprocessing the array. Prefix sums turn repeated summing into one subtraction.

Find Pivot Index

Easy

Pattern: Prefix sum / running left sum.

The goal is to find an index where the sum of all numbers on the left is equal to the sum of all numbers on the right. That index is called the pivot index.

My first working version

My solution first calculates the total sum of the array. Then, while scanning through the array, I keep track of the sum on the left side. The right side can be calculated using the total sum. 

class Solution:
              def pivotIndex(self, nums: list[int]) -> int:
                  total = 0
          
                  for el in nums:
                      total += el
          
                  sum_left = 0
          
                  for i in range(len(nums)):
                      rem = total - sum_left - nums[i]
          
                      if rem == sum_left:
                          return i
          
                      sum_left += nums[i]
          
                  return -1

This works because at each index, I can calculate the right-side sum without looping through the rest of the array. 

right_sum = total_sum - left_sum - current_number

If right_sum is equal to left_sum, then the current index is the pivot.

Improvement 1: Use clearer variable names

The logic is already good, but the variable name rem can be clearer. Since it represents the sum on the right side, I renamed it to right_sum. 

class Solution:
              def pivotIndex(self, nums: list[int]) -> int:
                  total_sum = 0
          
                  for num in nums:
                      total_sum += num
          
                  left_sum = 0
          
                  for i in range(len(nums)):
                      right_sum = total_sum - left_sum - nums[i]
          
                      if left_sum == right_sum:
                          return i
          
                      left_sum += nums[i]
          
                  return -1

Improvement 2: Use built-in sum() and enumerate()

Python already has sum() for calculating the total. Also, using enumerate() lets me loop over both the index and the value cleanly. 

class Solution:
              def pivotIndex(self, nums: list[int]) -> int:
                  total_sum = sum(nums)
                  left_sum = 0
          
                  for i, num in enumerate(nums):
                      right_sum = total_sum - left_sum - num
          
                      if left_sum == right_sum:
                          return i
          
                      left_sum += num
          
                  return -1

This is the cleanest version of the same idea. It is still doing exactly what my first solution did, just with clearer Python.

Complexity: O(n) time and O(1) extra space.

What I learned: I do not need to calculate the left and right sums from scratch for every index. If I know the total sum and keep a running left sum, I can get the right sum immediately.

Majority Element

Easy

Pattern: Sorting / hash map / Boyer-Moore voting.

The goal is to find the element that appears more than n / 2 times. This element is guaranteed to exist.

My first working version

My first solution sorts the array and returns the middle element. This works because the majority element appears more than half of the time, so after sorting it must pass through the middle position. 

class Solution:
              def majorityElement(self, nums: list[int]) -> int:
                  elems = sorted(nums)
                  return elems[len(nums) // 2]

For example, if 2 is the majority element, then after sorting, there are so many 2s that one of them must be located at the middle index.

Complexity of my first version: O(n log n) time because of sorting, and O(n) space because sorted(nums) creates a new list.

Improvement 1: Sort in place

If I am allowed to modify the input array, I can sort it in place. The logic stays the same, but I avoid creating a separate sorted copy. 

class Solution:
              def majorityElement(self, nums: list[int]) -> int:
                  nums.sort()
                  return nums[len(nums) // 2]

This is still O(n log n) time, but it avoids the extra list created by sorted().

Improvement 2: Use a hash map / Counter

Another approach is to count how many times each number appears. Once a number appears more than half of the array length, it is the majority element. 

from collections import Counter
          
          class Solution:
              def majorityElement(self, nums: list[int]) -> int:
                  counts = Counter(nums)
                  majority_limit = len(nums) // 2
          
                  for num, count in counts.items():
                      if count > majority_limit:
                          return num

This version is easy to understand because it directly follows the definition of a majority element: the value with count greater than n / 2.

Complexity: O(n) time and O(n) space.

Improvement 3: Boyer-Moore voting algorithm

The most efficient approach is the Boyer-Moore voting algorithm. The idea is to keep one candidate and a counter. If the counter becomes zero, I choose the current number as the new candidate.

Intuitively, different numbers cancel each other out. Since the majority element appears more than all other elements combined, it survives until the end. 

class Solution:
              def majorityElement(self, nums: list[int]) -> int:
                  candidate = None
                  count = 0
          
                  for num in nums:
                      if count == 0:
                          candidate = num
          
                      if num == candidate:
                          count += 1
                      else:
                          count -= 1
          
                  return candidate

This works because the problem guarantees that a majority element exists. If that guarantee did not exist, I would need one extra pass to verify that the candidate really appears more than n / 2 times.

Complexity of the final version: O(n) time and O(1) extra space.

What I learned: Sorting gives a very simple solution because the majority element must occupy the middle position. Counting makes the logic explicit. Boyer-Moore is the strongest version because it finds the answer in one pass using constant space.

Longest Consecutive Sequence

Medium

Pattern: Set lookup / sequence starting points.

The goal is to find the length of the longest sequence of consecutive numbers. The sequence does not need to appear in order in the original array.

My first working version

My solution converts the list into a set, so I can check whether a number exists in O(1) average time. Then, for each number, I only start counting if that number is the beginning of a sequence. 

class Solution:
              def longestConsecutive(self, nums: list[int]) -> int:
                  nums = set(nums)
                  max_count = 0
          
                  for number in nums:
                      if number - 1 not in nums:
                          y = number + 1
          
                          while y in nums:
                              y += 1
          
                          max_count = max(max_count, y - number)
          
                  return max_count

This works because a number is the start of a sequence only if the previous number does not exist. For example, 1 is the start of 1, 2, 3, 4 because 0 is not in the set.

Improvement 1: Use clearer variable names

The logic is already strong, but the variable name y is not very descriptive. I can rename it to current_num or next_num to make the code easier to read. 

class Solution:
              def longestConsecutive(self, nums: list[int]) -> int:
                  num_set = set(nums)
                  longest = 0
          
                  for num in num_set:
                      if num - 1 not in num_set:
                          next_num = num + 1
          
                          while next_num in num_set:
                              next_num += 1
          
                          longest = max(longest, next_num - num)
          
                  return longest

Improvement 2: Track current length explicitly

Another readable version is to keep a separate current_length variable. This makes the counting logic more obvious. 

class Solution:
              def longestConsecutive(self, nums: list[int]) -> int:
                  num_set = set(nums)
                  longest = 0
          
                  for num in num_set:
                      if num - 1 not in num_set:
                          current_num = num
                          current_length = 1
          
                          while current_num + 1 in num_set:
                              current_num += 1
                              current_length += 1
          
                          longest = max(longest, current_length)
          
                  return longest

This version says the idea very directly: start from the first number in a sequence, move forward while the next number exists, and count how long the sequence is.

Complexity: O(n) time and O(n) space. Even though there is a nested while loop, each number is only visited as part of a sequence once, because counting starts only from sequence beginnings.

What I learned: The important trick is not just using a set. The real trick is only starting from numbers that have no previous neighbour. That prevents counting the same sequence again and again.
